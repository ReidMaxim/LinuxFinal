Question 2:  

Describe in detail the differences between the kill, killall, and pkill commands, including common flags and arguments for each. Then demonstrate how to terminate a process using killall. Submit your description, alongside the commands you used.   

My answer: 

kill terminates a process using its PID 

common flags include: 

-9 = force kill. 

-l = list all available signal names.  

-2 = interrupt, similar to a ctrl+c in terminal.  

  

killall kills processes by name instead of PID  

  example: killall -9 firefox 

common flags include: 

  -i = prompts for confirmation before terminating.  

  -v =  verbose, provides a report on if the signal was sent to the target.  

  -l = list all available signal names 

  

pkill kills processes by pattern match. 

  common flags include: 

  -x = exact name  

  -u = kill process by user 

  -9 = force kill

Question 9: 

Describe the Linux boot processes from power on to desktop environment in exacting detail given the following: UEFI, drakut/systemdboot, systemd, gdm/gnome. 

 

Power is turned on and the CPU begins running the UEFI firmware. 

UEFI checks the hardware, performs startup tests, and searches for a bootable operating system. 

  

Once the correct boot entry is found, UEFI loads the bootloader into memory and hands over control. 

In this case, the bootloader is systemd-boot. 

  

systemd-boot reads its configuration files and displays the available boot options if needed. 

It then loads the Linux kernel and the initial ramdisk (initramfs) into memory. 

  

Control is passed to the Linux kernel. 

The kernel initializes system memory, processors, and hardware devices such as storage, USB, and graphics. 

  

Next, the initramfs created by dracut runs. 

Its job is to prepare the real root filesystem by loading drivers, detecting storage devices, and unlocking encrypted drives if necessary. 

  

After the root filesystem is ready, the system switches to the main operating system environment. 

The systemd init system starts as process ID 1 and begins launching essential background services. 

  

systemd mounts filesystems, starts networking, logging, device management, and other required services. 

It then reaches the graphical target, which starts the display manager. 

  

The display manager used here is GDM (GNOME Display Manager). 

GDM starts the graphical login screen and allows the user to sign in. 

  

After login, GDM launches the GNOME desktop session. 

GNOME starts the desktop shell, panels, settings services, and user applications.
